1. 为什么选择STM32

1.1 什么是STM32

  • 是新的基于ARM内核的32位的MCU系列
    内核为ARM公司为要求高性能、低成本、低功耗的嵌入式应用专门设计的Cortex-M的内核
  • 超前的体系结构
    • 高性能
    • 低电压
    • 低功耗
    • 创新的内核以及外设
  • 简单易用、自由、低风险

1.2 Cortex内核

Coretex-M3系列属于ARMv7架构

ARMv7架构定义了三大分工明确的系列：
A系列：面向尖端的虚拟内存操作系统和用户应用 （eg. android）
R系列：针对实时系统
M系列：对微控制器

STM32F1属于Cortex-M系列中的Cortex-M3内核，采用ARMv7-M架构。STM32F4采用ARMv7-ME架构。Cortex-A5/A8采用ARMv7-A架构。
传统的ARM7系列采用的是ARMv4T架构。

STM32也属于ARM，但是是新架构。

1.3 CM3内核优点

  1. 高性能Cortex-M内核
  • 采用ARM公司流行的标准内核Cortex-M3
  • 地动态功耗上实现的高性能
  1. 最佳的代码密度
  • Thumb-2指令集以16位指令的密度实现32位指令性能（与ARM7TDMI的ARM模式比减少了30%-45%的代码量）
  1. 可预见的运行时间
  • 中断控制器嵌再内核值中，中断之间的间隔最少可达6个CPU周期。
  • 从低功耗模式唤醒只需要6个CPU周期。
  1. 改进的调试功能
  • 串行单步调试和JTAG调试

2.STM32概述

2.1 CM3芯片结构

Cortex-M3内核、调试系统：ARM设计
内部总线、外设、存储器、时钟和复位、I/O：由芯片制造商设计开发。

2.2 STM32F103系列

  • 2V-3.6V供电电压
  • 5V I//Os电压容限
  • 出色的时钟安全模式
  • 带有唤醒功能的低功耗模式
  • 内部RC
  • 嵌入的RESET
  • -40~+85℃（工业级）

一般区分芯片方式：闪存容量和引脚数目

2.3 STM32F1各芯片关联

强大的固件库支持 -> 各系列产品全面兼容
优点

  • 可升级的体系结构
  • 使用所有应用
  • 给用户相同的“外观和感受”
  • 统一的软件和开发投入

2.4 STM32命名规则

实例： STMF103C8T6
STM32 F 103 C 8 T 6

  • STM32 : 产品系列。
    • STM32 = 基于ARM的32位微控制器
  • F : 产品类型。
    • F = 通用类型
  • 103 : 产品子系列。
    • 101 = 基本型；
    • 102 = USB基本型，USB 2.0全速设备；
    • 103 = 增强型；
    • 105或107 = 互联型
  • C : 引脚数目。
    • T = 36脚；
    • C = 48脚；
    • R = 64脚；
    • V = 100脚；
    • Z = 144脚
  • 8 : 闪存存储器容量
    • 4 = 16K字节的闪存存储器；
    • 6 = 32K字节的闪存存储器；
    • 8 = 64K字节的闪存存储器；
    • B = 128K字节的闪存存储器；
    • C = 256K字节的闪存存储器；
    • D = 384K字节的闪存存储器；
    • E = 512K字节的闪存存储器
  • T : 封装
    • H = BGA
    • T = LQFP
    • U = VFQFPN
    • Y = WLCSP64
  • 6 : 温度范围
    • 6 = 工业级温度范围 -40~85℃
    • 7 = 工业级温度范围 -40~105℃

3.总结

STM32应用场景

  • 销售节点机
  • USB设备
  • 工业自动化
  • 建筑安防、消防
  • 低功耗
  • 家电
  • …
    本质就是高性能控制器

STM32最具竞争力的优势

  • 极高的性能：主流Cortex内核
  • 丰富合理外设、合理功耗和价格
  • 强大软件支持：丰富的软件包
  • 丰富的技术文档
  • 芯片型号种类多，覆盖面广
  • 强大用户基础
